Returns the listener array for the specified event. Will initialise the event object and listener arrays if required. Will return an object if you use a regex search.   • remove
  • evt
  • [listeners]
)
Object

Defined in src\service\events.js:309

Edits listeners in bulk. The addListeners and removeListeners methods both use this to do their job. You should really use those instead, this is a little lower level. The first argument will determine if the listeners are removed (true) or added (false). If you pass an object as the second argument you can add/remove from multiple events at once. The object should contain key value pairs of events and listeners or listener arrays. You can also pass it an event name and an array of listeners to be added/removed. You can pass it a regular expression to manipulate the listeners of all events that match it.

Parameters:

  • remove Boolean

    True if you want to remove listeners, false if you want to add.

  • evt String | Object | RegExp

    An event name if you will pass an array of listeners next.

  • [listeners] Function optional

    An optional array of listener functions to add/remove.

Returns:

Object:

)
Object

Defined in src\service\events.js:395

Emits an event of your choice. When emitted, every listener attached to that event will be executed. If you pass the optional argument array then those arguments will be passed to every listener upon execution. Because it uses apply, your array of arguments will be passed as if you wrote them out separately. So they will not arrive within the array on the other side, they will be separate. You can also pass a regular expression to emit to all events that match it.

Parameters:

  • evt String | RegExp

    Name of the event to emit and execute listeners for.

  • [args] Array optional

    Optional array of arguments to be passed to each listener.
